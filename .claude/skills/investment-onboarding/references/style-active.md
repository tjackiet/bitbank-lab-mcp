# アクティブ（波に乗って売買する）

テクニカル分析やデータ分析でエッジを見つけ、相場の波を捉えて機動的に売買する運用スタイル。

ユーザーには「アクティブ」という内部名称ではなく、
**「相場の波を捉えて売買する方針で進めますね」** のように行動表現で伝える。

> 注：このガイドは将来的に専用 Skill `investment-active-trading` に拡張される予定。
> 現時点では本ファイルでアクティブ／レバレッジ両方の入口をカバーする。

---

## 戦術候補

### ① トレンドフォロー（順張り）

トレンドの方向に乗って、流れが続く限り保有する。**初心者にはここから案内する。**

- SMA クロス（ゴールデンクロス／デッドクロス）
- MACD クロス
- 一目均衡表の雲抜け
- RSI でエントリータイミングを絞る（トレンド中の押し目）

### ② 逆張り・リバーサル

行き過ぎた動きの反転を狙う。中級者向け（相場状況の判断が必要）。

- RSI 買われすぎ／売られすぎ
- ボリンジャーバンドの反発
- MACD ダイバージェンス
- サポート／レジスタンスでの反発

### ③ ブレイクアウト

レンジを抜けた方向についていく。中級者向け。

- ボリンジャーバンドのスクイーズ → ブレイクアウト
- サポート／レジスタンスの突破
- 出来高増加を伴うブレイク

### ④ オルタナティブデータ分析

外部データやオンチェーンデータからエッジを探して売買する。上級者向け。

- ステーブルコインフロー、Coinbase プレミアム、ETF 資金流入／流出、
  オンチェーンデータ等
- bitbank API だけでは完結しない（外部データソースが必要）

---

## 戦術の選び方

ユーザーに次のように尋ねる：

> 売買のアプローチは大きく 4 つあります。どれが近いですか？
>
> 1. **トレンドに乗って取る** — 流れが続く限り保有する（初心者向け）
> 2. **行き過ぎの反発を狙う** — 反転を狙う（中級者向け）
> 3. **レンジ抜けを取る** — ブレイクアウトについていく（中級者向け）
> 4. **外部データでエッジを探す** — オンチェーン等の独自データを使う（上級者向け）
> 5. **おすすめで進める**

「おすすめ」「わからない」が選ばれた場合は **① トレンドフォロー** で進める。

## ワークフロー

```text
1. 環境分析     — 市場全体の状態を把握
2. 銘柄チェック  — 対象銘柄の現在の指標を読む
3. 戦略評価     — バックテストで仮説を検証
4. 実行        — 注文する
5. モニタリング  — 結果を確認・改善
```

### Step 1 — 環境分析

「今、相場全体はどんな状態か」を把握する。トレンド相場かレンジ相場かで
有効な戦術が変わる。

**MCP ツール:**
- `analyze_market_signal` — 5 要素を -100〜+100 のスコアで瞬時評価。**最初の一手**
- `get_volatility_metrics` — ATR・実現ボラティリティ等
- `get_flow_metrics` — CVD・アグレッサー比

### Step 2 — 銘柄チェック

対象銘柄の現在の指標を読む。

**MCP ツール（戦術別の使い分け）:**

| 戦術 | 主に使うツール |
|---|---|
| ① トレンドフォロー | `analyze_indicators`, `analyze_sma_snapshot`, `analyze_ema_snapshot`, `analyze_ichimoku_snapshot`, `analyze_mtf_sma`, `detect_macd_cross` |
| ② 逆張り | `analyze_stoch_snapshot`, `analyze_bb_snapshot`, `analyze_support_resistance`, `analyze_indicators`（RSI 用途） |
| ③ ブレイクアウト | `analyze_bb_snapshot`（スクイーズ検出）, `analyze_support_resistance`, `analyze_volume_profile`, `detect_patterns` |
| ④ オルタナティブ | `get_flow_metrics`, `detect_whale_events`, `analyze_volume_profile` + 外部データ |

ローソク足パターンの確認には `analyze_candle_patterns` を補助的に使う。
チャートパターン（ダブルトップ等）は `detect_patterns` で検出。

### Step 3 — 戦略評価（バックテスト）

「この戦略は過去データで機能しているか」を確認する。
裁量で売買する場合もスキップせず一度は回すことを推奨。

**MCP ツール:**
- `run_backtest` — SMA / MACD / BB / RSI 戦略のバックテストを一括実行
  - データ取得 → 計算 → チャート描画まで自動

### Step 4 — 実行

**MCP ツール（Private API キー必要）:**
- `preview_order` → `create_order` の 2 ステップ確認
  - 必ず `preview_order` で内容確認してから `create_order` を呼ぶ

逆指値・指値の使い分けはユーザーに任せる。
損切りラインを事前に決めておくことを案内する。

### Step 5 — モニタリング

アクティブは頻度高めの確認が必要（週 1 〜 日次）。

**MCP ツール:**
- `get_my_assets` / `analyze_my_portfolio` — 損益・構成比
- `get_my_orders` — 未約定注文の確認
- `get_my_trade_history` — 約定履歴の振り返り

## ポリシー

- **特定銘柄を推奨しない。** 判断軸とツールの分析結果を見せ、選択は委ねる。
- **トレンドフォローを最初の選択肢として案内する。** 逆張り・ブレイクアウト・
  オルタナティブは状況判断や追加データが必要。
- **損切りラインを事前に決めることを案内する。** アクティブはエントリーよりも
  撤退ルールが重要。
- **バックテストの結果は過去のもの。** 将来の成果を保証しないことを
  明記する。
