# レバレッジ（信用取引で大きく狙う）

アクティブの戦術に信用取引（買い建玉／売り建玉）を組み合わせ、
リスク・リターンを増幅する運用スタイル。

ユーザーには「レバレッジ」という内部名称ではなく、
**「信用取引を使って大きめに狙う方針で進めますね」** のように行動表現で伝える。

---

## 構造

レバレッジは独自の戦術体系を持つというよりも、
**アクティブの戦術に信用取引が加わる構造**。

```
レバレッジ = アクティブの戦術（①〜④）× 信用取引（買い建玉／売り建玉）
```

戦術の選び方・ワークフローは [style-active.md](style-active.md) を参照する。
このガイドでは **信用取引固有の要素** に絞って案内する。

---

## bitbank の信用取引の制約

| 項目 | 内容 |
|---|---|
| 最大レバレッジ | 2 倍（日本の規制） |
| オプション取引 | 取扱なし |
| 取引方向 | 買い建玉（ロング）／売り建玉（ショート）両方可能 |
| ロスカット | あり。証拠金維持率の管理が必須 |

レバレッジ 2 倍という制約上、1 回の取引で大きく勝つよりも、
**信用取引 × アクティブ戦術を高頻度で回してリターンを積み上げる方向** に
なりやすい点を案内する。

---

## 信用取引で広がる選択肢

| 要素 | 内容 |
|---|---|
| ロングにレバレッジ | リターンもリスクも増幅される |
| ショートが使える | 下落局面でも利益を狙える |
| ヘッジ | 現物保有の下落リスクを軽減できる |

## ワークフロー

アクティブのワークフローに加えて、信用取引固有のステップが入る。

```
1. 環境分析     — アクティブと同じ
2. 銘柄チェック  — アクティブと同じ
3. 戦略評価     — アクティブと同じ（必ずバックテストを回す）
4. ポジション設計 — 建玉方向・レバレッジ・証拠金を決める
5. 実行         — preview_order（position_side 指定）→ create_order
6. ポジション管理 — 証拠金維持率・ロスカットラインの確認
7. 決済         — 利確／損切り
8. モニタリング  — 損益・建玉状況の確認
```

### Step 1〜3 — アクティブと共通

[style-active.md](style-active.md) の Step 1〜3 を参照する。

### Step 4 — ポジション設計

信用取引に特有のステップ。次を事前に決めておく：

- **建玉方向** — ロング（買い）かショート（売り）か
- **レバレッジ倍率** — 1.0 倍〜2.0 倍の範囲で決める
- **証拠金** — どの程度の証拠金を入れるか
- **損切りライン** — どこまで逆行したら決済するか
- **利確ライン** — どこまで進んだら決済するか

> ⚠️ ロスカットを食らう前にユーザー自身で損切りすることを基本ルールとして案内する。

### Step 5 — 実行

**MCP ツール（Private API キー必要）:**
- `preview_order`（`position_side: long` または `short` を指定）→ `create_order`

### Step 6 — ポジション管理

建玉を持っている間は定期的な監視が必要。

**MCP ツール:**
- `get_margin_status` — 証拠金維持率・建玉サマリー
- `get_margin_positions` — 各建玉の詳細

維持率が低下している場合は、追加証拠金を入れるか早めに決済するかを判断する。

### Step 7 — 決済

利確・損切りのいずれも、**事前に決めたラインを守る** ことを案内する。

**MCP ツール:**
- `preview_order` → `create_order`（決済方向で発注）

### Step 8 — モニタリング

**MCP ツール:**
- `get_margin_trade_history` — 信用約定履歴
- `analyze_my_portfolio` — 全体の損益・構成
- `get_my_assets` — 残高・利用可能証拠金

---

## ポリシー

- **特定銘柄・建玉方向を推奨しない。** 判断軸とツールの分析結果を見せ、
  選択は委ねる。
- **損切りラインを事前に決めることを必ず案内する。** レバレッジ取引では
  撤退ルールが資金を守る最重要の規律。
- **ロスカットリスクを毎回ではなく初回に明示する。** 「証拠金維持率を割ると
  強制決済される」点を理解してもらう。
- **アクティブの戦術選択はそのまま使える。** トレンドフォロー／逆張り／
  ブレイクアウト／オルタナティブの分類は変わらない。
