# グロース（伸びる銘柄を狙う）

ファンダメンタルズやモメンタムを分析し、有望な銘柄を選んで値上がりを狙う運用スタイル。

ユーザーには「グロース」という内部名称ではなく、
**「有望銘柄を選んで値上がりを狙う方針で進めますね」** のように行動表現で伝える。

---

## 戦術候補

### ① ファンダメンタルズで選ぶ

プロジェクトの実体・需給構造から有望銘柄を見つける。中級者以上向け。

- **構造的な需給** — トークノミクス、発行上限、バーン、アンロックスケジュール、
  ステーキングロック等。プロトコル設計として決まっているもの
- **プロジェクトの実態** — 用途・ユースケース、開発活動（ロードマップ進捗）、
  事業進捗（提携、資金調達、新サービス、上場、アップデート等）、
  採用実績、コミュニティの強さ

### ② モメンタムで選ぶ

市場で実際に動いている銘柄を見つける。**初心者にはここから案内する。**
MCP ツール群との親和性が最も高い。

- 通貨強弱（市場全体に対して相対的に強い銘柄を見つける／順張り）
- 市場の需給（出来高推移、大口の動き、資金流入/流出、ポジション偏り）

### ③ テーマで選ぶ

伸びる領域に張る。中級者以上向け。

- DeFi、AI、ゲーム、L1/L2 等のテーマで銘柄を選ぶ
- 同テーマ内で割安な銘柄を見つける（逆張り）
- bitbank の取扱銘柄に限られるため、選択肢は限定的

---

## 戦術の選び方

ユーザーに次のように尋ねる：

> 銘柄選びのアプローチは大きく 3 つあります。どれが近いですか？
>
> 1. **市場で動いている銘柄を見つける** — 価格・出来高の動きから探す（初心者向け）
> 2. **プロジェクトの実態で選ぶ** — トークン設計や開発状況を調べる（中級者向け）
> 3. **テーマで選ぶ** — DeFi・AI・ゲーム等のジャンルから探す（中級者向け）
> 4. **おすすめで進める** — 最初の選択肢で進める

「おすすめ」「わからない」が選ばれた場合は **② モメンタム** で進める。

---

## ワークフロー

```text
1. 方針を決める（①②③のどのアプローチで選ぶか）
2. 銘柄を絞り込む
3. 買い時を判断する
4. 買う
5. 定期レビュー（偏りチェック・リバランス）
```

### Step 1 — 方針を決める

上記「戦術の選び方」で確定する。

### Step 2 — 銘柄を絞り込む

**② モメンタムを選んだ場合（推奨フロー）:**

| 観点 | MCP ツール |
|---|---|
| 通貨強弱ランキング | `analyze_currency_strength`（複合スコアで全 JPY ペアを評価） |
| 市場全体のシグナル | `analyze_market_signal`（5 要素を -100〜+100 のスコアに集約） |
| 出来高ランキング | `get_tickers_jpy`（24h 変化率・出来高で並べ替え） |
| 大口の動き | `detect_whale_events` |
| 資金フロー | `get_flow_metrics`（CVD・アグレッサー比） |
| 価格帯別出来高 | `analyze_volume_profile` |

**最初の一手：** `analyze_currency_strength` を呼び、
強い順から数銘柄をピックアップしてユーザーに見せる。

**① ファンダメンタルズを選んだ場合:**

- MCP ツールだけでは完結しない。bitbank の公式情報、各プロジェクトの公式サイト、
  ホワイトペーパー、X（旧 Twitter）等の情報源を案内する。
- 構造的な需給（発行上限・バーン）は事実として確認しやすい。
- 事業進捗・採用実績はユーザー自身のリサーチが必要。

**③ テーマを選んだ場合:**

- bitbank の取扱銘柄を `get_tickers_jpy` で一覧化し、
  ユーザーが知りたいテーマでフィルタする（例：L1 系、DeFi 系）。
- 同テーマ内で時価総額・出来高に乖離がある場合は、割安候補として扱える。

### Step 3 — 買い時の判断

絞り込んだ銘柄について、エントリータイミングを判断する。

**MCP ツール:**
- `analyze_indicators` — RSI / MACD / BB / 一目 / SMA を一括で確認
- `analyze_support_resistance` — サポート水準で押し目を探す
- `analyze_fibonacci` / `analyze_mtf_fibonacci` — リトレースメント水準

### Step 4 — 購入

**MCP ツール（Private API キー必要）:**
- `preview_order` → `create_order` の 2 ステップ確認

### Step 5 — 定期レビュー

グロースは銘柄選定の偏りが出やすい。月 1 回程度のレビューを推奨。

**MCP ツール:**
- `get_my_assets` / `analyze_my_portfolio` — 構成比と損益を確認
- `get_tickers_jpy` — 候補から外したい銘柄・新たに加えたい銘柄を比較

---

## ポリシー

- **特定銘柄を推奨しない。** 判断軸とツールの分析結果を見せ、選択は委ねる。
- **モメンタムを最初の選択肢として案内する。** ファンダ・テーマは中級者向け。
- **ファンダ分析は MCP だけでは完結しない。** ユーザー自身のリサーチが必要な点を
  正直に伝える。

> 興味があればのぞいてみてください。

同一セッション内で 2 回目以降は出さない。
